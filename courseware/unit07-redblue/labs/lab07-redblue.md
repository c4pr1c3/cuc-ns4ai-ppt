# 实验 07：红蓝对抗与综合复盘（综合实践项目 M7）

> 本实验即综合实践项目（capstone） **M7**，全学期收口。在「已加固 + 自带 AI 能力 + 已可观测」的集成系统上跑一次红蓝演练 → 取证复盘 + **能力框架自评矩阵**。本里程碑「自评」权重上调到 0.20（回看整学期 M0–M6），并设**现场演示**。

## 1. 实验目标

- **能力交付（绑定全簇 · L3 · 综合工程化）**:
  1. 能够编排一场红蓝对抗（侦察 → 渗透 → AI 对抗 → 防御 → 取证闭环）。
  2. 能够在演练中综合运用 M0–M6 全部能力，并度量纵深防御。
  3. 能够产出**能力框架自评矩阵**（逐簇 × 级回看 M0–M6 证据，无虚点亮）。
  4. 能够评估 AI 在攻防中的综合表现与局限（簇⑥·L3）。

> 完成本实验后，在 `docs/m7/report.md` 自评：点亮**全簇 · L3**。M7 评审会**回看 M0–M6**——自评矩阵每项都要能指回仓库证据。

## 2. 环境准备

- **本地**：你的 `milestone/m6` 分支 tip（M6 末系统：已加固 + AI 能力 + 加固证据 + 集中日志）。
- 按 [Git 规范](../../unit00-intro/labs/git-guide.md) **从 `milestone/m6` 切 `milestone/m7`**，MR 目标 = `milestone/m6`。
- **角色**：分组 / 自扮红蓝；攻击面 = 你自己的系统。
- **学术诚信**：仅在自己靶场 / 授权环境；不教授编写恶意代码。

## 3. 任务清单

### 任务 A：红蓝对抗场景与规则设计
设计演练场景（攻击面 = 自己的系统）、目标 / 规则 / 停战条件、评分维度；明确仅授权环境。

### 任务 B：红队编排
复用 M3 / M6 的概念验证（PoC）编排一条多阶段攻击链（侦察 → 渗透 → AI 对抗 / 提示词注入 → 横向），记录每步。

### 任务 C：蓝队防御复盘
检测 / 取证 / 欺骗闭环（复用 M4 / M5）：哪些被拦、哪些漏检、纵深防御各层命中率；度量防御纵深度。

### 任务 D：能力框架自评矩阵（核心）
对照 [`capability-framework.md`](../../../capability-framework.md) 的 6 簇 × 3 级，逐项回看 M0–M6 证据，标注是否点亮 + 证据路径 + **本里程碑练了哪个思维范式**。覆盖 **≥3 个簇**，无虚点亮。

### 任务 E：AI 综合评估（簇⑥·L3）
评估 AI 在本次演练中的综合表现：赋能侧（检测 / 分诊）+ 对抗侧（AI 被攻破的路径）+ **局限**（什么场景 AI 失效 / 误报）。

### 任务 F：现场演示
现场跑一次红蓝对抗演示（或提交**录屏 / 动态证据包**——动态证据静态无法核验，不交录屏则降级 observation）。

### AI 助教适配注意事项

- **常见扣分**：自评虚点亮（点了簇×级但找不到 M0–M6 证据）、覆盖 < 3 簇、无现场演示 / 录屏、缺 AI 局限分析。
- **授权红线**：演练仅在自己系统；触及第三方 = 安全严谨不合格 + 上报。
- **AI 双向**：M7 的 AI 双向按「是否在演练中综合评估 AI（攻防 + 局限）」给档（权重 0.20）。
- **自评矩阵**：本里程碑自评权重 0.20（最高之一），逐簇 × 级附证据，「无虚点亮」是硬要求。
